Интерактивное веб-приложение для анализа эффективности маркетинговых каналов и оптимизации распределения рекламного бюджета с использованием статистического моделирования Marketing Mix Modeling (MMM).

## 🎯 Описание

Marketing Mix Model — это статистический подход для измерения влияния различных маркетинговых каналов на бизнес-метрики и оптимизации распределения рекламного бюджета. Система учитывает:

- **Эффект переноса (Adstock)** — влияние рекламы на будущие периоды
- **Эффект насыщения (Saturation)** — убывающую отдачу от рекламных вложений
- **Внешние факторы** — сезонность, конкуренцию, экономические показатели
- **Синергию каналов** — взаимодействие между различными видами рекламы

## 📊 Основные возможности

### 📈 Анализ атрибуции

- "What-if" анализ изменений в медиа-планах

## 📊 Формат входных данных

Приложение ожидает CSV файл со следующими столбцами:

### Обязательные столбцы:
- `date` — дата (YYYY-MM-DD)
- `orders/sales/revenue` — целевая метрика

### Медиа-каналы (расходы):
- `facebook_spend` — расходы на Facebook Ads
- `google_spend` — расходы на Google Ads
- `tiktok_spend` — расходы на TikTok Ads
- `youtube_spend` — расходы на YouTube Ads
- `offline_spend` — офлайн реклама

### Внешние факторы (опционально):
- `promo_activity` — промо-активность (0/1)
- `competitor_activity` — активность конкурентов (0-2)
- `holiday_effect` — праздничные эффекты (0/1)

### Пример данных:
```csv
date,orders,facebook_spend,google_spend,tiktok_spend,promo_activity
2023-01-01,5847,45000,67000,12000,0
2023-01-08,6123,48000,71000,15000,1
2023-01-15,5892,42000,69000,18000,0
```

## 🧮 Математические основы

### Основная модель MMM:
```
Sales(t) = Base + Σ[Adstock(Media_i) × Saturation(Media_i)] + External_factors(t) + ε(t)
```

### Adstock (эффект переноса):
```
Geometric: Adstock(t) = Media(t) + λ × Adstock(t-1)
где λ ∈ [0,1] - коэффициент затухания
```

### Saturation (эффект насыщения):
```
Hill: Saturation = Media^α / (Media^α + γ^α)
где α - форма кривой, γ - точка полунасыщения
```

### Оптимизация бюджета:
```
maximize: f(Sales, ROAS, ROI)
subject to: Σ(Budget_i) = Total_Budget
           Min_i ≤ Budget_i ≤ Max_i
```

## 📈 Использование приложения

### 1. Загрузка данных
- Используйте **демо-данные** для быстрого тестирования
- Загрузите **собственный CSV файл** с историческими данными
- Проверьте **валидацию данных** для выявления проблем качества

### 2. Конфигурация модели
- Выберите **целевую метрику** (заказы, продажи, выручка)
- Определите **медиа-каналы** для анализа
- Настройте **параметры Adstock** (decay rate, max lag)
- Задайте **параметры Saturation** (alpha, gamma)

### 3. Обучение модели
- Установите **долю обучающей выборки** (рекомендуется 80%)
- Выберите **тип регуляризации** (Ridge, Lasso, ElasticNet)
- Запустите **обучение модели** и оцените качество (R², MAPE)

### 4. Анализ результатов
- Изучите **декомпозицию продаж** по каналам
- Проанализируйте **ROAS и предельный ROAS**
- Просмотрите **кривые насыщения** для каждого канала
- Оцените **качество модели** на тестовых данных

### 5. Оптимизация бюджета
- Задайте **общий бюджет** для распределения
- Установите **ограничения по каналам** (мин/макс)
- Выберите **цель оптимизации** (продажи, ROAS, ROI)
- Получите **оптимальное распределение** бюджета

### 6. Сценарное планирование
- Создайте **собственные сценарии** распределения
- Сравните **предустановленные стратегии**:
  - Digital Focus (фокус на онлайн каналы)
  - Balanced (равномерное распределение)
  - Performance (фокус на высокоэффективные каналы)

## 📈 Интерпретация результатов

### Метрики качества модели:
- **R² > 0.7** — хорошее качество модели
- **MAPE < 15%** — приемлемая точность прогноза
- **Остатки близки к нормальному распределению** — корректность модели

### ROAS интерпретация:
- **ROAS > 3** — высокоэффективный канал
- **1 < ROAS < 3** — умеренно эффективный
- **ROAS < 1** — убыточный канал (требует пересмотра)

### Предельный ROAS:
- Показывает эффективность **дополнительного рубля**
- Используется для **перераспределения бюджета**
- Каналы с высоким Marginal ROAS — приоритет для увеличения

## ⚠️ Ограничения MVP

### Технические ограничения:
- **Единый файл** — все классы в app.py для упрощения развертывания
- Данные за **12-104 недели** (оптимально 2 года)
- **Weekly агрегация** (недельные данные)
- Упрощенная модель **без взаимодействий** между каналами

### Предположения модели:
- **Линейная аддитивность** эффектов каналов
- **Стационарность** базовой линии продаж
- **Постоянные внешние факторы** в прогнозном периоде
- Отсутствие **канибализации** между каналами
